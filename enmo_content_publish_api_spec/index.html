Mutations

createContent

Response

Returns a CreateContentRespGql!

Arguments
Name Description
appId - Int! Enmo Application ID where the content is being published.
multiMediaContent - String! Keep this empty.
basicDetails - CreateContentReqGqlInput!

Example

Query
mutation createContent(
  $appId: Int!,
  $multiMediaContent: String!,
  $basicDetails: CreateContentReqGqlInput!
) {
  createContent(
    appId: $appId,
    multiMediaContent: $multiMediaContent,
    basicDetails: $basicDetails
  ) {
    contentId
  }
}
Variables
{
  "appId": 123,
  "multiMediaContent": "xyz789",
  "basicDetails": CreateContentReqGqlInput
}
Response
{"data": {"createContent": {"contentId": 987}}}

Types

CreateContentReqGqlInput

Fields
Input Field Description
backgroundColor - String! Background color of the post. Use HEX value of the color. Eg: #195ed4
backgroundGraphic - String!
contentType - String! Use TEXT_TEMPLATE
description - String! Small descriptive paragraph about your content.
enableClickToCall - Boolean! Keep false.
enableComments - Boolean! Whether to enable comments for this post. (true/false)
enableLikes - Boolean! Whether to enable likes for this post. (true/false)
fromDate - String! From which date the content should be published. Use yyyy-mm-dd format. Eg: 2022-10-01. Keep empty to publish immediately.
status - String! User approval_not_required.
templateText - String! Content of the post.
textColor - String! Text color of the post. You have to use the color name. Supported Color Names
toDate - String! Till when the content will be valid. Use yyyy-mm-dd format. Eg: 2022-10-01
valid - Boolean! Use true.
Example
{
  "backgroundColor": "#195ed4",
  "backgroundGraphic": "",
  "contentType": "TEXT_TEMPLATE",
  "description": "Enmo Rich Content application will help to get your information in rich and colorful manner.",
  "enableClickToCall": false,
  "enableComments": true,
  "enableLikes": true,
  "fromDate": "2022-10-01",
  "status": "approval_not_required",
  "templateText": "Enmo Rocks",
  "textColor": "Crimson",
  "toDate": "2022-10-30",
  "valid": true
}
